## Appendix A: Execution Context Rules

### TARG Scoping
Each `conn.execute()` runs in its own context. TARG is NULL between calls.
Batch dependent commands in one call using newline separation.

### Variable Persistence
- `VA00`–`VA99`: Local to script execution. Lost between execute() calls.
- `OV00`–`OV99`: Persist on the agent object. Survive between calls if agent lives.
- `GAME "varname"`: Persist globally across all scripts and sessions.
- `NAME "varname"`: Persist on TARG agent (string variables).

### Script vs Injection Context
- `SCRP family genus species event`: Register a persistent script in the scriptorium.
- `JECT "filename.cos" flags`: Inject/run a COS file.
- `conn.execute("command")`: Injection context via shared memory: no OWNR, TARG starts NULL.

### Shared Memory Protocol
- Maximum command length: ~4096 bytes (shared memory buffer size).
- Mutex timeout: engine holds mutex during execution.
- Result timeout: if CAOS takes too long, the bridge gets RESULT TIMEOUT.
- Newline batching: separate multiple commands with `\n` in a single execute() call.

### INST/SLOW/LOCK
- `INST`: Run following commands without yielding. Required for atomic operations.
- `SLOW`: Return to normal yielding execution.
- `LOCK`: Prevent script interruption. Paired with `UNLK`.

```
targ [creature]
```

```
targ [creature]
```

```
targ [creature]
setv va00 0

  addv va00 1
untl va00 eq 20
```
